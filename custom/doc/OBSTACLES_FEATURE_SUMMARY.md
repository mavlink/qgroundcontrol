# Конспект: препятствия в паттерне Spray (QGroundControl custom)

Краткое описание реализации зон препятствий (no-fly) с буфером и обводкой по границе, чтобы другой разработчик мог повторить функционал.

---

## 1. Цель

- Пользователь рисует **поле** (один полигон) и опционально **препятствия** (несколько полигонов внутри поля).
- Трансекты (линии сканирования) не заходят внутрь препятствий и не заходят в **буфер** вокруг них (параметр **ObstacleBuffer**, метры).
- При перелёте между двумя обрезанными сегментами маршрут идёт **вдоль границы препятствия** (boundary path), а не напрямую через него.
- На карте видны: само препятствие (красное) и **зона буфера** (оранжевая полупрозрачная), чтобы было видно «запрещённый» отступ.

---

## 2. Структура данных (C++)

- **Поле:** `QGCMapPolygon _sprayAreaPolygon` — один полигон.
- **Препятствия:** `QmlObjectListModel* _obstaclePolygonsModel` — список `QGCMapPolygon*`. Каждое препятствие — отдельный полигон, минимум 3 точки.
- **Буфер:** `SettingsFact _obstacleBufferFact` (имя в JSON: `"ObstacleBuffer"`), единицы — метры, по умолчанию 1.5. Используется только для расчётов и отображения зоны буфера.
- **Расширенные полигоны для карты:** `QList<QList<QGeoCoordinate>> _obstacleBufferPolygons` — результат расширения каждого препятствия на величину буфера (в геокоординатах), обновляется в `_rebuildTransects` и отдаётся в QML через свойство `obstacleBufferPolygons` (QVariantList).

Подключение к пересчёту:

- При изменении полигона поля, буфера, списка препятствий или вершин любого препятствия вызывается `_rebuildTransects()`.

---

## 3. Алгоритм (все расчёты в NED относительно поля)

**Tangent origin:** первая точка полигона поля  
`QGeoCoordinate tangentOrigin = _sprayAreaPolygon.pathModel().value<QGCQGeoCoordinate*>(0)->coordinate();`  
Все полигоны и линии переводятся в локальную NED-сетку (x = East, y = North) относительно этой точки через `QGCGeo::convertGeoToNed` / `convertNedToGeo`.

### 3.1. Построение «разрешённой» области (Clipper2)

1. Поле и препятствия конвертируются в NED → `QPolygonF polygon` (поле), `QList<QPolygonF> obstaclesNed` (препятствия; полигоны замкнуты, последняя точка = первая).
2. Каждое препятствие **расширяется** на `ObstacleBuffer` метров:
   - `Clipper2Lib::InflatePaths(..., margin, JoinType::Round, EndType::Polygon, 2.0, decimalPrec)`.
3. Все расширенные полигоны объединяются в один (Union) → `unionObstacles`.
4. **Разрешённая область** = поле минус объединённые расширенные препятствия:  
   `Difference(fieldPaths, unionObstacles)`.
5. Результат Clipper2 (PathsD) разбивается на **внешний контур** и **дыры**:
   - внешний контур — контур с максимальной площадью по модулю (само поле);
   - остальные контуры — дыры (места препятствий с буфером).  
   Функция `allowedPathsToOuterAndHoles`.

Важно: для Clipper2 полигоны должны иметь **положительную площадь** (обход против часовой стрелки в обычной математической системе). Используется `ensurePositiveArea(path)` — при отрицательной площади путь разворачивается.

### 3.2. Обрезка линий трансектов

- Каждая линия трансекта (от пересечения с полем до пересечения с полем) обрезается по разрешённой области функцией **`_clipLineByAllowedArea(line, allowedOuter, allowedHoles)`**.
- Возвращается список отрезков `QList<QLineF>`, лежащих внутри внешнего контура и вне всех дыр.
- Реализация: пересечение линии с внешним полигоном, затем вычитание пересечений с каждой дырой; оставшиеся отрезки — допустимые сегменты.

### 3.3. Boundary path (обводка препятствия между сегментами)

Когда два последовательных допустимых сегмента не стыкуются (между концом одного и началом следующего есть разрыв — между ними препятствие):

1. **`_findObstacleCrossing(p1, p2, obstaclesNed, obstacleIndex, cross1, cross2)`**  
   Определяет, какое препятствие пересекает отрезок p1–p2, и возвращает две точки пересечения (cross1, cross2) в порядке от p1 к p2.

2. **`_boundaryPathBetweenPoints(cross1, cross2, polygonObstacle, targetHint)`**  
   Строит путь вдоль **оригинального** (не расширенного) полигона препятствия от cross1 до cross2.  
   - **targetHint** — следующая точка маршрута (начало следующего сегмента), чтобы выбрать дугу обхода **в сторону целевой точки** и не заворачивать лишний раз вокруг препятствия.
   - Точки cross1/cross2 лежат на рёбрах полигона; проверка «точка на ребре» делается по условию:  
     `dist(vertex_i, p) + dist(p, vertex_{i+1}) <= length(edge) + eps`  
     с использованием **длины ребра** и **обеих длин** до точки (не только одной), иначе на острых углах возможны ошибки.
   - Дуга выбирается сравнением расстояний от следующей вершины вперёд/назад до targetHint.

3. Полученные NED-точки boundary path конвертируются в геокоординаты и вставляются в трансект между концом текущего сегмента и началом следующего.

## 4. Визуализация (QML)

- **SprayMapVisuals.qml** (карта):
  - Поле: `QGCMapPolygonVisuals` для `sprayAreaPolygon`.
  - **Буферы препятствий:** Repeater по `_missionItem.obstacleBufferPolygons`, для каждого элемента — `MapPolygon` с `path: modelData`, цвет/граница оранжевые, полупрозрачная заливка (например `#44ff8800`), чтобы было видно зону отступа.
  - **Сами препятствия:** Repeater по `_missionItem.obstaclePolygons`, для каждого — `QGCMapPolygonVisuals` (тёмно-красная граница и заливка).
- Порядок рисования: сначала буферы, потом препятствия, чтобы контур препятствия был поверх буфера.

C++ должен отдавать `obstacleBufferPolygons` как `QVariantList`: каждый элемент — список геокоординат одного расширенного полигона. Список заполняется в `_rebuildTransects` после расчёта `expandedObstacles` (до Union): каждый PathD из Clipper2 конвертируется в `QList<QGeoCoordinate>` через `convertNedToGeo(path[i].y, path[i].x, 0, tangentOrigin, coord)` и добавляется в `_obstacleBufferPolygons`. В конце блока с препятствиями вызывается `emit obstacleBufferPolygonsChanged()` (при отсутствии препятствий список пустой, сигнал тоже нужен).

---

## 5. Настройки и UI

- **Spray.SettingsGroup.json:** параметр `ObstacleBuffer` (double, м, min 0, по умолчанию 1.5), shortDesc про «no-fly margin».
- **SprayItemEditor.qml:** слайдер/поле для `missionItem.obstacleBuffer`, секция «Obstacles (no-fly zones)» с кнопками «Add obstacle» / удаление по индексу, список `missionItem.obstaclePolygons`.

---

## 6. Сохранение/загрузка (JSON)

- Версия complex item: 3.
- В корень объекта сохраняются: `obstacleBuffer` (число), массив `obstacles` — для каждого препятствия вызывается `obst->saveToJson(obstObj)` (стандартное сохранение полигона QGCMapPolygon).
- При загрузке: чтение `obstacleBuffer`, по умолчанию 1.5; для каждого элемента массива `obstacles` создаётся `QGCMapPolygon`, загрузка через `loadFromJson`, при успехе — connect на `pathChanged` → `_rebuildTransects` и добавление в `_obstaclePolygonsModel`.

---

## 7. Нюансы и подводные камни

1. **Система координат:** везде в расчётах NED (x — East, y — North). В Clipper2 и в `QPolygonF` точки в порядке (x, y). В `QGCGeo::convertNedToGeo` порядок аргументов (y, x, down, ...).

2. **Замкнутость полигонов:** при конвертации в PathD дублирующую последнюю точку (равную первой) можно не добавлять; при обратной конвертации pathDToPolygon при необходимости дописывать замыкание.

3. **Обход границы препятствия:** без `targetHint` и выбора дуги «к цели» маршрут может пойти длинной дугой вокруг препятствия. Всегда передавать в `_boundaryPathBetweenPoints` в качестве targetHint точку начала следующего сегмента.

4. **Точка на ребре:** проверять принадлежность точки отрезку через сумму длин (от концов до точки) и длину ребра, с небольшим eps. Использовать длину именно **ребра** и оба расстояния в условии, иначе на краях ребра возможны пропуски/неверный выбор ребра.

5. **Clipper2:** библиотека подключается через CPM (см. custom/CMakeLists.txt). Точность для операций задаётся (например, decimalPrec = 2). InflatePaths может вернуть несколько контуров для одного препятствия (например, при сильном расширении) — все их нужно учитывать и в Union, и в списке для отображения буфера.

6. **Обновление буфера на карте:** `_obstacleBufferPolygons` и сигнал `obstacleBufferPolygonsChanged` должны вызываться при каждом пересчёте трансектов (и когда препятствий нет — список очищается, сигнал всё равно испускается).

7. **Добавление препятствия:** при создании нового препятствия (например, квадрат 20×20 м вокруг центра поля) не забыть подписать его на `pathChanged` и `dirtyChanged` и добавить в модель; при загрузке из JSON — только `pathChanged` на `_rebuildTransects`.
